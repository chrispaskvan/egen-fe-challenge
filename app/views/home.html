About My Egen Solutions Angular Front End Challenge

Click the "Sign Up" button to see the Password Strength Angular Directive in action.

Features
  • I took this opportunity to experiment with the Semantic UI responsive design layout.
  • I wrote 2 directives for the Popup and Progress Semantic components. The progress component receives input from the SignUp Controller as the controller manages the feedback provided by the PasswordStrength directive.
  • I ran out of time, but I would have liked to have wired in the Users service to mock a service request to verify the provided user name is unique. This is common practice for me.
  • I wanted the PasswordStrength directive to be a attribute only directive. I often have to integrate with different design frameworks such as Bootstrap. It's often easier for the web designer to change frameworks or adjust layouts if the markup not somewhere else. Using the attributes on the input, I can modify the behavior of the directive without necessarily changing the markup or using any specific CSS framework in a template.
  • To demonstrate the implementation of the attributes, I required more than 1 of a couple of different qualifiers. This ability to reuse the directive by configuring it with attributes is also a common requirement for me as clients often have different password requirements and changing those requirements is not an option.
  • I included the ngForm directive taken from Mark Pieszak. I often cannot use form tags or assume form tags are available in my integrations with client websites. So I tend to avoid depending on the existence of a form tag with a submit action. The ngForm mimics this tag and allows the enter key to cause validation.
  • I always try to follow John Papa's Angular 1 Style Guide.
